Quick Start — Basic Cache Setup

Before diving into multi-provider fallback chains and subagents, here's the foundation. One API key, one model, cache enabled. This alone can cut your API costs by 90%. Everything else on this page builds on top of this.

4 things. That's it.
1
Get an Anthropic API key
Go to console.anthropic.com → API Keys → Create Key. Add credit ($5 minimum). That's your ANTHROPIC_API_KEY.
2
Set cache_control on your system prompt
Add cache_control: { type: "ephemeral" } to your system message. This tells Anthropic to cache that block for up to 1 hour. Every subsequent request that sends the same system prompt reads from cache at 0.1× the cost of fresh tokens.
3
Keep your system prompt stable
Cache keys are based on exact content. If you change even one character in your system prompt, it's a cache miss and you pay full price. Put anything that changes (user messages, timestamps) after the cached block, not inside it. Stable prefix = cache hits = 90% savings.
4
Set a heartbeat to keep the cache alive
The cache TTL is 1 hour. If you go 60 minutes without sending a request, the cache expires silently and your next request pays full price again. Set a lightweight ping every 30 minutes — even a minimal request with the same system prompt is enough. This keeps the cache warm through gaps in activity. Without it, long thinking breaks or idle periods will quietly kill your savings.

💡 The first request costs ~2× because it writes to cache. Every request after that — for the next hour — reads from cache at 0.1× cost. If you send 100 requests in an hour, you paid full price once and got 99 at 90% off. The longer your system prompt, the bigger the savings. A 10,000 token system prompt cached across 50 requests saves roughly $4.50 per hour versus uncached.
📐 That's the entire basic setup. One API key. One model. The cache_control block on your system prompt. A 30-minute heartbeat to keep it warm. Everything below this point — fallback chains, multiple providers, subagents, context pruning — is optimisation built on top of this foundation.

Authentication Profiles

OpenClaw supports multiple provider profiles simultaneously. Intermittent connectivity issues with the Anthropic Direct API resolved by using Openrouter as alternative provider. Openrouter configuration is matched to Anthropic models only. Openrouter is more expensive per token than Anthropic Direct, therefore use it as fallback for intermittend connectivity issues with Direct API. Each profile maps a provider name to an auth mode. Agents reference these profiles via the provider/model syntax in their model config.

Setting Value What it does
bootstrapMaxChars 25000 Maximum characters loaded from memory/context on agent startup. Keeps initial context lean and fast.
contextPruning.mode cache-ttl Prune context entries that have fallen out of the cache window. Keeps the context window consistent with what's actually cached — no point sending tokens that aren't cached.
contextPruning.ttl 1h Match the Anthropic ephemeral cache TTL exactly. Context older than 1 hour is pruned since the cache for it will have expired anyway.
compaction.mode safeguard When context gets very long, safeguard mode compacts conservatively — preserving critical information and avoiding lossy summarisation that could corrupt agent state.
heartbeat.every 30m Agent pings every 30 minutes to keep the cache alive and detect stale connections before they fail mid-task.
memorySearch.provider voyage Voyage AI embeddings for semantic memory search. When an agent needs to recall past context, Voyage finds the most relevant chunks rather than doing keyword matching.
🧠 cache-ttl pruning + ephemeral cache = self-consistent context. The pruning TTL is deliberately set to match the cache TTL. This means the context window always reflects what's cached — you never pay full price for tokens whose cache has already expired, and you never have stale references in context pointing to evicted cache entries.

Concurrency Model

🤖 Main Agents
4
Parallel top-level agents running simultaneously. Each runs a full primary + fallback model chain. Suitable for independent workstreams that don't need to share state.
Primary: claude-sonnet-4-6
Via: Anthropic direct → OpenRouter → Opus → ...
⚡ Subagents
8
Parallel subagents per main agent. Delegated tasks run on free/cheap models — Nemotron-120B free tier first. Subagents inherit the task but use a separate, cost-optimised model chain.
Primary: nemotron-3-super-120b:free
Fallback: Gemini Flash → Qwen3-235B → Sonnet 4.5
⚡ The subagent model chain is deliberately cost-inverted from the main agent chain. Main agents use the best Anthropic models. Subagents start with a free 120B model and only escalate to paid models if that fails. This means delegated subtasks cost near-zero while the primary reasoning layer stays high-quality.
